css水平与垂直居中和多列布局

发表于 2016-07-01

css布局之水平垂直居中和多列布局

  • 水平垂直居中

    子元素于父元素垂直及水平居中且其（子元素与父元素）高度宽度均可变。

  • inline-block + text-align + table-cell + vertical-align

此方法的好处在于兼容性好。

  • absolute + transform

此方法的好处在于绝对定位脱离文档流，不会对后续元素的布局造成影响，但transform 为 CSS3 属性，有兼容性问题。

  • flex + justify-content + align-items

此方法的好处在于只需设置父节点属性，无需设置子元素,但存在兼容性问题。

多列布局

多列布局在网页中非常常见（例如两列布局），多列布局可以是两列定宽，一列自适应， 或者多列不定宽一列自适应还有等分布局等。

  • 一列定宽，一列自适应

  • float + margin

此方法IE 6 中会有3像素的 BUG，解决方法可以在 .left 加入 margin-left:-3px。

  • float + margin + (fix) 改造版

此方法不会存在 IE 6 中3像素的 BUG，但 .left 不可选择， 需要设置 .left {position: relative} 来提高层级。 此方法可以适用于多版本浏览器（包括 IE6）。缺点是多余的 HTML 文本结构。

  • float + overflow

此方法在设置 overflow: hidden 会触发 BFC 模式（Block Formatting Context）块级格式化文本。BFC 中的内容与外界的元素是隔离的。优点在于样式简单，但不支持IE6。

  • table

table 的显示特性为每列的单元格宽度合一定等与表格宽度。 table-layout: fixed; 可加速渲染，也是设定布局优先。注意：table-cell 中不可以设置 margin 但是可以通过 padding 来设置间距。

  • flex

flex-item 默认为内容宽度。在低版本浏览器兼容问题，存在性能问题，只适合小范围布局。

  • 两列定宽，一列自适应

多列定宽的实现可以更具单列定宽的例子进行修改与实现。

css布局

发表于 2016-06-28

css布局之垂直居中

子元素于父元素垂直居中且其（子元素与父元素）宽度均可变

  • table-cell + vertical-align

此方法的好处在于兼容性好（支持 IE8，以下版本需要调整页面结构至 table）。

  • absolute + transform

此方法的好处在于绝对定位元素脱离文档流，不会对后续元素的布局造成影响。但如果绝对定位元素是唯一的元素则父元素也会失去高度，但transform为CSS3属性，存在兼容性问题。

  • flex + align-items

此方法的好处在于只需设置父节点属性，无需设置子元素,但存在兼容性问题。

css水平布局

发表于 2016-06-19

css布局之水平居中

子元素于父元素水平居中且其（子元素与父元素）宽度均可变

  • inline-block + text-align

此方法的好处在于兼容性好（甚至可以兼容 IE 6 和 IE 7）。

  • absolute + transform

此方法的好处在于使用绝对定位后元素会脱离文档流，不会对后续元素的布局造成影响。

但要注意的是transform 属于CSS3属性，存在兼容性问题.

  • table + margin

display: table 在表现上类似 block 元素，但是宽度为内容宽,无需设置父元素样式 （支持 IE 8 及其以上版本）,兼容 IE 8 以下版本则需要调整table。

  • flex + justify-content

这种方法的好处在于只需设置父节点属性，无需设置子元素，但此种方法存在兼容性问题。
