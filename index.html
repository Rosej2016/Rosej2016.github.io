css布局之多列等分布局

发表于 2016-07-04

css布局之多列等分布局，每一列的宽度和间距均相等，下面为多列等分布局的布局特定。

  • float

此方法可以完美兼容 IE8 以上版本。 NOTE+：此方法结构和样式具有耦合性。

  • table

此方法的缺点是多了文本结果。

  • flex

flex 的特性为分配剩余空间。 NOTE+：兼容性有问题。

  • table

table 的特性为每列等宽，每行等高可以用于解决此需求。

  • flex

flex 默认的 align-items 的值为 stretch。

  • float

此方法为伪等高（只有背景显示高度相等），左右真实的高度其实不相等,此方法兼容性较好。

css水平布局

发表于 2016-06-29

css布局之水平居中

子元素于父元素水平居中且其（子元素与父元素）宽度均可变

  • inline-block + text-align

此方法的好处在于兼容性好（甚至可以兼容 IE 6 和 IE 7）。

  • absolute + transform

此方法的好处在于使用绝对定位后元素会脱离文档流，不会对后续元素的布局造成影响。

但要注意的是transform 属于CSS3属性，存在兼容性问题.

  • table + margin

display: table 在表现上类似 block 元素，但是宽度为内容宽,无需设置父元素样式 （支持 IE 8 及其以上版本）,兼容 IE 8 以下版本则需要调整table。

  • flex + justify-content

这种方法的好处在于只需设置父节点属性，无需设置子元素，但此种方法存在兼容性问题。

css布局

发表于 2016-06-28

css布局之垂直居中

子元素于父元素垂直居中且其（子元素与父元素）宽度均可变

  • table-cell + vertical-align

此方法的好处在于兼容性好（支持 IE8，以下版本需要调整页面结构至 table）。

  • absolute + transform

此方法的好处在于绝对定位元素脱离文档流，不会对后续元素的布局造成影响。但如果绝对定位元素是唯一的元素则父元素也会失去高度，但transform为CSS3属性，存在兼容性问题。

  • flex + align-items

此方法的好处在于只需设置父节点属性，无需设置子元素,但存在兼容性问题。
